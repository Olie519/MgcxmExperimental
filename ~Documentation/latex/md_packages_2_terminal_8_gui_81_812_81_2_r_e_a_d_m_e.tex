\begin{DoxyItemize}
\item {\bfseries{Cross Platform}} -\/ Windows, Mac, and Linux. Terminal drivers for Curses, \href{https://github.com/gui-cs/Terminal.Gui/issues/27}{\texttt{ Windows Console}}, and the .NET Console mean apps will work well on both color and monochrome terminals.
\item {\bfseries{Keyboard and Mouse Input}} -\/ Both keyboard and mouse input are supported, including support for drag \& drop.
\item {\bfseries{\href{https://gui-cs.github.io/Terminal.Gui/articles/overview.html\#layout}{\texttt{ Flexible Layout}}}} -\/ Supports both {\itshape Absolute layout} and an innovative {\itshape Computed Layout} system. {\itshape Computed Layout} makes it easy to layout controls relative to each other and enables dynamic terminal UIs.
\item {\bfseries{Clipboard support}} -\/ Cut, Copy, and Paste of text provided through the \href{https://gui-cs.github.io/Terminal.Gui/api/Terminal.Gui/Terminal.Gui.Clipboard.html}{\texttt{ {\ttfamily Clipboard}}} class.
\item {\bfseries{\href{https://gui-cs.github.io/Terminal.Gui/api/Terminal.Gui/Terminal.Gui.View.html}{\texttt{ Arbitrary Views}}}} -\/ All visible UI elements are subclasses of the {\ttfamily View} class, and these in turn can contain an arbitrary number of sub-\/views.
\item {\bfseries{Advanced App Features}} -\/ The \href{https://gui-cs.github.io/Terminal.Gui/api/Terminal.Gui/Terminal.Gui.MainLoop.html}{\texttt{ Mainloop}} supports processing events, idle handlers, timers, and monitoring file descriptors. Most classes are safe for threading.
\item {\bfseries{Reactive Extensions}} -\/ Use \href{https://github.com/dotnet/reactive}{\texttt{ reactive extensions}} and benefit from increased code readability, and the ability to apply the MVVM pattern and \href{https://www.reactiveui.net/}{\texttt{ Reactive\+UI}} data bindings. See the \href{https://github.com/gui-cs/Terminal.Gui/tree/master/ReactiveExample}{\texttt{ source code}} of a sample app in order to learn how to achieve this.
\end{DoxyItemize}\hypertarget{md_packages_2_terminal_8_gui_81_812_81_2_r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{\texorpdfstring{Showcase \& Examples}{Showcase \& Examples}}\label{md_packages_2_terminal_8_gui_81_812_81_2_r_e_a_d_m_e_autotoc_md5}

Run {\ttfamily dotnet run -\/-\/project UICatalog} to run the UI Catalog.

\item {\bfseries{\href{https://github.com/gui-cs/Terminal.Gui/tree/master/FSharpExample}{\texttt{ F\# Example}}}} -\/ An example showing how to build a Terminal.\+Gui app using F\#.
\item {\bfseries{\href{https://github.com/gui-cs/Terminal.Gui/tree/master/ReactiveExample}{\texttt{ Reactive Example}}}} -\/ A sample app that shows how to use {\ttfamily System.\+Reactive} and {\ttfamily Reactive\+UI} with {\ttfamily Terminal.\+Gui}. The app uses the MVVM architecture that may seem familiar to folks coming from WPF, Xamarin Forms, UWP, Avalonia, or Windows Forms.
